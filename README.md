# Region-Based License Plate Localization Using Morphological and Edge-Based Processing

## Overview

This project implements a lightweight, classical image processing pipeline for automatic license plate localization in Java. The system does not rely on machine learning or deep learning models. Instead, it applies a structured sequence of morphological and edge-based operations to identify and extract license plate regions from vehicle images.

The approach is designed to be computationally efficient and deployable in resource-constrained environments while maintaining acceptable localization accuracy.

## Processing Pipeline

The localization pipeline consists of the following sequential stages:

### 1. Grayscale Conversion

The input RGB image is converted to a single-channel 8-bit grayscale image using a luminance-weighted formula:

```
I(x, y) = 0.2126R + 0.7152G + 0.0722B
```

This reduces computational overhead and removes colour information that is not required for edge and shape analysis.

### 2. Noise Reduction

A median filter operating on a 3x3 neighbourhood is applied to the grayscale image. For each pixel, the intensity values within the neighbourhood are sorted and replaced by the median value. This suppresses salt-and-pepper noise while preserving edge boundaries that are critical for plate detection.

### 3. Edge Detection

The Sobel operator is applied to compute horizontal and vertical intensity gradients:

```
Gx = [-1  0  1]     Gy = [ 1  2  1]
     [-2  0  2]           [ 0  0  0]
     [-1  0  1]           [-1 -2 -1]
```

The resulting gradient magnitude image highlights strong intensity transitions that correspond to object boundaries including the borders of the license plate.

### 4. Binarization

Otsu's thresholding method is used to automatically determine an optimal threshold from the image histogram. The method maximises the inter-class variance between foreground and background pixel distributions, producing an adaptive binary image that is robust to illumination differences across the dataset.

### 5. Morphological Closing

A binary morphological closing operation is applied using a 3x3 structuring element:

```
I * B = (I ⊕ B) ⊖ B
```

Dilation bridges small gaps between fragmented edge segments. Erosion then removes spurious expansions introduced during dilation. The combined operation consolidates disconnected plate edges into a single coherent foreground region suitable for connected component analysis.

### 6. License Plate Localization

Connected component analysis using an 8-connected neighbourhood identifies distinct foreground regions in the binary image. A tight bounding box is computed for each region. Geometric filtering based on area and aspect ratio (width-to-height ratio) is applied to discard non-plate candidates. The region best satisfying the geometric constraints is selected as the license plate and its bounding box is projected onto the original colour image for visualisation.

## Dataset

The evaluation dataset consists of 97 vehicle images provided under controlled experimental conditions. Images vary in illumination, background complexity and capture angle.

---

## Results

| Metric | Value |
|---|---|
| Total test images | 97 |
| Successfully localized | 72% |
| Processing time per image | ~967 ms |

Detection failures were primarily observed in images with severe illumination variation, low contrast or highly cluttered backgrounds.

## Requirements

- Java Development Kit (JDK) 8 or later
- Eclipse IDE (recommended, based on project structure) or any standard Java IDE
- No external libraries or machine learning frameworks are required

---

## How to Run

1. Open the project in Eclipse by selecting **File > Open Projects from File System** and pointing to the project root directory.
2. Ensure the `images/` directory contains the vehicle images to be processed.
3. Build the project. Eclipse will compile source files from `src/` into `bin/`.
4. Run the main class from the `src/` directory.
5. The program will process each image through the pipeline and output the localized plate with a bounding box overlay.

---

## Limitations

- Performance degrades under severe illumination variation or low-contrast conditions where edge information becomes weak or inconsistent.
- Fixed geometric thresholds for area and aspect ratio limit adaptability to non-standard plate formats.
- The system does not perform character recognition; it localizes the plate region only.

---

## Future Work

- Adaptive contrast enhancement to improve robustness under varying lighting conditions
- Dynamic geometric constraints that adjust to image resolution
- Integration of a lightweight classification stage to reduce false positives in cluttered scenes
